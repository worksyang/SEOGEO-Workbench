# WritingMoney 批量成稿批次制重构计划

## 1. 产品结论

母文章铸造和批量成稿不是同一种“项目”：

- **母文章铸造**管理长期内容资产。一个项目对应一个母关键词和一篇可持续迭代的母文章。
- **批量成稿**管理一次生产任务。一个项目对应一个批次，批次内包含一组关键词、AI 匹配的母文章、推荐钩子、生成篇数和成稿队列。
- 两个模式共享母文章库，但分别拥有独立的左侧列表、独立的新建逻辑和独立的状态。

批量成稿不再以某一篇母文章为中心，而是以**关键词为中心**。一个关键词可以匹配零篇、一篇或多篇母文章；一篇母文章也可以服务多个关键词。

## 2. 从旧系统继承什么

旧系统的真实组合是：

`源 Markdown + TXT 中抽取的标题 + 统一 Prompt -> AI 成稿 -> Markdown 后处理 -> 输出目录`

新系统保留其中有效的生产思维：

- 一次运行先形成一个明确批次。
- 每个成稿任务都必须有关键词、母文章输入和提示词上下文。
- 同一母文章可以循环服务多个关键词。
- 批次可以并发运行，并保留成功、运行中、等待、失败/返工状态。
- 写作系统只输出 Markdown 和批次清单；发布系统通过目录或 API 消费结果。

不继承旧系统的随机黑箱、Windows 固定目录、发布账号管理、Selenium 发布和业务脏词替换。

## 3. 页面结构

### 3.1 系统导航

仍保留两个模式：

- 母文章铸造
- 批量成稿

切换模式时只替换项目列表和工作区，母文章铸造现有视觉与业务流程保持不变。

### 3.2 批量模式左侧列表

母文章模式显示“文章项目”；批量模式改为显示“成稿批次”。

批次列表字段：

- 批次名称
- 来源：手动创建 / 选题监控
- 计划成稿数
- 状态：待确认 / 生成中 / 已完成
- 更新时间

批次按“今天 / 本周 / 已完成”分组。顶部按钮改为“新建批次”，搜索框改为“搜索批次 / 来源 / 状态”。

### 3.3 批量模式工作区

只保留两个步骤：

1. **批次配置**
2. **成稿队列**

## 4. 新建批次

新建批次弹窗包含：

- 批次名称
- 本批次写作要求：允许输入自然语言，例如“今天分红实现率是热点，相关关键词多写几篇”
- 关键词：每行一个，可由选题系统未来通过接口传入
- 输出目录

点击“创建并智能匹配”后，Demo 模拟完成三件事：

1. 按语义为每个关键词匹配零篇、一篇或多篇母文章。
2. 根据热点描述为每个关键词推荐生成篇数。
3. 为每个关键词匹配一个引流钩子。

创建结果进入“待确认”批次，而不是立即开始写作。

## 5. 批次数据模型

### 5.1 Batch

```js
{
  id,
  name,
  source,
  brief,
  status,
  createdAt,
  updatedAt,
  outputDir,
  publishHandoff,
  keywords
}
```

### 5.2 KeywordTask

```js
{
  id,
  keyword,
  signal,
  signalReason,
  purpose,
  count,
  recommendedCount,
  hookId,
  motherMatches,
  readiness
}
```

### 5.3 MotherMatch

```js
{
  motherId,
  role,
  confidence
}
```

`readiness` 只表达生产前置条件：

- `ready`：已有母文章，可以直接批量成稿。
- `needs-mother`：没有合适母文章，需要回母文章铸造。

不设置“质量通过”“重复度检查”“事实校验”等当前尚未真实存在的伪状态。

## 6. 批次配置页

页面顶部显示批次名称、来源、自然语言要求、更新时间和输出目录。

概览只显示四个真正有决策价值的数字：

- 关键词数
- 计划成稿数
- 可直接生成
- 缺母文章

关键词队列以关键词为主行，字段如下：

| 字段 | 用途 |
|---|---|
| 关键词 | 当前生产任务的基准 |
| 热点信号 | 普通 / 热点 / 高优先，解释推荐篇数 |
| 匹配母文章 | 支持 0、1、多篇，并显示各自作用 |
| 推荐钩子 | 每个关键词独立推荐 |
| 生成篇数 | 默认由 AI 分配，用户可加减至 0 |
| 操作 | 编辑写作目的、调整匹配、去铸造母文章 |

交互规则：

- 用户可以修改关键词和单独写作目的。
- 用户可以增减篇数，最低为 0。
- 用户可以移除已有母文章匹配，或从母文章库补选。
- 缺母文章的关键词允许保留在批次中，但不会进入成稿队列。
- “去铸造母文章”需要确认后切换到母文章模式。
- 点击“确认批次，生成队列”后进入成稿队列。

## 7. 成稿队列

成稿队列按 `关键词 × 生成篇数` 展开，每一行是一篇真实待生成文章。

字段：

- 预生成标题
- 命中关键词
- 使用的母文章
- 引流钩子
- 状态：等待 / 运行中 / 已完成 / 需返工
- 输出文件
- 操作：查看、重跑、回母文章铸造

顶部概览：

- 计划生成
- 等待
- 运行中
- 已完成
- 需返工

Demo 点击开始后模拟状态变化，但不伪装成真实 LLM 调用。

## 8. 与选题系统和发布系统的边界

### 8.1 选题系统

未来只需要向 WritingMoney 提交：

```json
{
  "batch_name": "2026-07-12 分红实现率热点",
  "brief": "今天分红实现率是热点，相关关键词提高覆盖",
  "keywords": []
}
```

WritingMoney 负责母文章匹配、篇数建议、钩子推荐和成稿。

### 8.2 发布系统

发布逻辑继续解耦。WritingMoney 只交付：

- 批次目录
- 成稿 Markdown
- `batch_manifest.json`

未来发布系统可以读取目录，也可以通过批次接口获取 manifest。Demo 只显示“发布交接：目录输出，接口未连接”，不加入账号、排期和发布按钮。

## 9. Demo 数据

至少准备三个独立批次：

1. **分红实现率热点追踪**：来自选题监控，包含热点关键词，部分关键词默认 2-3 篇。
2. **提领现金流长尾补位**：手动创建，大部分关键词一篇。
3. **产品横评专项**：包含一个关键词匹配两篇母文章的示例。

同时覆盖：

- 一对多：一篇母文章服务多个关键词。
- 多对一：一个关键词需要两篇母文章。
- 零匹配：关键词缺少母文章，需要回铸造。
- 热点加量：单关键词默认生成多篇。

## 10. 验收标准

- 切换母文章/批量模式时，左侧列表内容和标题同时切换。
- 母文章铸造现有样式与流程不被改动。
- 批量模式的新建按钮创建的是批次，不是文章项目。
- 批次列表和母文章项目列表互不混用。
- 每个关键词支持 0/1/多篇母文章匹配。
- 每个关键词支持独立写作目的、钩子和生成篇数。
- 缺母文章能明确返回母文章铸造，并有确认弹窗。
- 成稿队列按实际生成篇数展开。
- 桌面和移动端均无溢出、遮挡或不可点击控件。
- 浏览器逐项测试模式切换、新建批次、编辑关键词、加减篇数、母文章匹配、进入队列、返工跳转。
