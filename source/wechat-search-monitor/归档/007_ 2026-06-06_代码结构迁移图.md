## 本轮迁移重点

1. 不改变页面效果。
2. 不改变 `normalized/*.json` 输出结构。
3. 不改变 `python3 scripts/parse_search_md.py` 的使用方式。
4. 只把“职责太重的单脚本”拆成更清晰的模块。

## 迁移完成后的职责边界

### 1. Parser

- 输入：原始 Markdown
- 输出：`ParsedSnapshot`
- 不负责落盘
- 不负责账号/文章/快照实体聚合
- 不负责前端视图计算

### 2. Entity Builder

- 输入：`ParsedSnapshot[]`
- 输出：`keywords/snapshots/articles/...`
- 负责事实层构建
- 不负责页面趋势和评分

### 3. Monitor Builder

- 输入：事实层实体
- 输出：`monitor-data.json`
- 负责热度分、账号分、15 天趋势、前端视图结构

### 4. Rebuild Orchestrator

- 负责串联：
  - discover
  - parse
  - build entities
  - build monitor data
  - write json
- 是 CLI 入口背后的真正执行者

### 5. Web 层

- 只读取结构化结果
- 只处理 API 和状态变更
- 不直接内嵌解析逻辑
