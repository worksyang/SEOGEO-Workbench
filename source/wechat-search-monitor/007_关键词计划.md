# 关键词计划

## 灵感一：热点锚词驱动的 SEO 选题机制

### 痛点

监控系统现在已经能知道"哪些账号排上来了、哪些词我们没上榜"，但**不知道该写什么才能上榜**。

现有的选题逻辑是被动的——看别人写了什么，再跟着做。缺少一个主动信号：**在竞争对手还没大量跟进之前，找到那个能让产品词快速上榜的组合词**。

核心问题在于：微信搜索的排名逻辑里，单纯写一篇"安盛盛利2测评"已经很难冒出来，因为同类内容太多，向量空间里早已饱和。

### 底层机制（已验证的直觉）

微信搜索本质上是一个**新鲜度敏感的语义检索系统**。它的偏好：

1. **高权重锚词加持**：语料库中天然高频、高关注度的词（如"美联储""600亿""苹果"），携带这类词的文章会被判定为"高关注度语境"，排名天然上浮。

2. **新向量对奖励**：原本没有共现关系的两个词，突然在短时间内被多个账号同时使用（如"信守明天 + 600亿"），系统会判断这是新鲜事件，优先推出来。

3. **窗口期存在**：新鲜共现词的红利窗口通常只有 48-72 小时，之后内容饱和，红利消失。

**核心公式：**
> 产品词（要排名的） + 高权重锚词（借势） + 新鲜共现词（热点） = 快速上榜

### 解决方案

在现有 `ranking_hits.title_raw` 数据上，新增一层**词频差分分析**：

- 统计过去 72 小时内，头部账号标题中出现的词频
- 与过去 7 天的基线词频做差分
- 增量最大的词 = 当前热点锚词候选

每天输出一批"热点锚词 × 产品词"的组合建议，作为优先选题。已经上榜的产品词跳过，未上榜的词优先匹配热点锚词，争取窗口期内发文。
