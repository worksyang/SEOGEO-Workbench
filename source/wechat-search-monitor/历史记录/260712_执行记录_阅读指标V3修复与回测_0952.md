# 阅读指标V3修复与回测

## 修改范围

本次只修改“常态阅读 / 15日阅读增量”的计算、质量字段、解释文字和专项回测，不改关键词刷新策略、调度器、探针逻辑、账号评分或其他业务代码。

## V3公式变化

1. **缺失日不再统一填中位数。** 改为相邻观测线性插值，并随距离向长期常态中位数收缩。
2. **刷新频率进入公式。** 每天/3天使用15日历史；7天使用28日历史；15天使用60日历史。
3. **新词成熟保护。** 首次成功观测未满15天时，不输出正式常态阅读和15日增量，不再向首次监控之前补出虚构历史。
4. **估算占比修正。** 缺失日期的补值现在全部计入 `estimated_share`；新增 `imputed_day_share`。
5. **关键词独立计算。** 单个关键词的数值不再因“同一批次还跑了多少其他关键词”而机械变化；跨词去重应留给词库总盘点，不进入单词比较指标。
6. **质量字段补齐。** 新增刷新频率、历史回看期、首次/末次观测、成熟天数、计划观测数、计划覆盖率、新鲜度和补值占比。
7. 方法版本从 `rank_standardized_expected_v2` 升级为 `schedule_adjusted_read_rate_v3`。

## 回测口径

- 历史参考样本：133个active稳定词，15日内至少13个高密度观测日。
- 把真实历史快照重新抽样为3天、7天、15天，再按V3重算。
- “达到60%准确率”的可复算定义：估算值落在高密度参考值的 **60%～140%** 范围内，即相对误差不超过40%。
- 同时记录排序相关度、可用率和中位误差；回测脚本：`scripts/backtest_keyword_read_metrics.py`。
