# AIDSO 爱搜 KeywordWSO 板块页面结构详解

> 本文档基于 2026-06-09 对 `https://dso.aidso.com/KeywordWSO/*` 全站关键词监控板块的逐页人工探索整理而成。
> 目标：让任何 Agent 仅凭本文档，即可对 KeywordWSO 板块的页面结构、数据字段、标签枚举、付费墙、单位格式、爬虫接入方式形成准确、可执行的认知。
> 探索覆盖了至少 15 个组合页面：单个/批量 × 4 个主 tab × 多个关键词（顺丰、友邦、蜜雪冰城、安盛盛利2、奶茶、伯牙等）× 左侧分词/全部两种视图。

## 一、板块定位与 URL 形态

KeywordWSO 是 AIDSO 爱搜大数据平台面向**微信搜一搜（WSO = WeChat Search Optimization）**的关键词分析模块。平台一级域名是 `dso.aidso.com`，WSO 子路径是 `/KeywordWSO/`。

主要页面 URL：

| 路径 | 用途 | 说明 |
|---|---|---|
| `/KeywordWSO/searchWord` | 关键词搜索主页（本次重点） | 单个/批量、挖掘/分词、4 tab 数据展示 |
| `/KeywordWSO/searchWord?keyword=奶茶` | 带预填关键词的搜索页 | URL 上的 `keyword` 参数会预填到输入框 |
| `/WsoKeyWordDetail/detail?keyword=友邦` | 单关键词详情页 | 已有的 Playwright 抓取入口 |

入口位置：登录后顶部导航「关键词」下拉 → 「微·关键词」或「WSO关键词」。

## 五、搜索模式切换与输入控件

紧贴频道选择条右上角的是**单个 / 批量**切换器。

### 5.1 单个模式（默认）

- 一个文本输入框
- placeholder 文本：`输入关键词，可多词联查，例:"服装+定制"`
- 支持 `+` 号连接多词查询
- 右侧有「挖掘」「分词」两个按钮

### 5.2 批量模式

- 切换后输入框变成 textarea / 列表式输入
- 占位提示：`模糊搜索 0/5 个人版最多输入5个词 文件上传`
- 个人版最多 5 个词；升级旗舰版/企业版可提升上限
- 支持「文件上传」批量导入关键词

### 5.3 操作按钮语义

- **挖掘**：执行搜索（默认行为）
- **分词**：执行分词模式（仅在分词/全部视图下可见）

> 重要：`挖掘` vs `分词` 是同一个输入框下的两个不同动作按钮，分词按钮会触发"分词视图"并填充左侧分词树。

## 七、4 个主 Tab（核心数据区）

右侧主区域的核心是 4 个 tab，每个 tab 是不同的数据视图。tab 元素 role=`tab`，DOM id 如下：

| Tab | DOM id | 默认状态 | 数据后端 |
|---|---|---|---|
| 搜索词 | `tab-searchWord` | **active** | 搜索词主索引 |
| 相关词 | `tab-correlationWord` | — | 相关词索引 |
| 下拉词 | `tab-sugWord` | — | 微信下拉词补全索引 |
| 行业词 | `tab-industryWord` | — | **付费墙拦截** |

### 7.1 搜索词 Tab（默认）

- 数据规模提示：「AIDSO爱搜共20w+个搜索结果」
- 表格列（6 列 + 操作）：

| # | 列名 | 字段语义 | 单位/格式 | 空值 |
|---|---|---|---|---|
| 1 | 微·关键词 | 关键词名称 | string | — |
| 2 | 月均搜索量 | 月均搜索热度 | string（带w）或 int | 0 |
| 3 | 月均点击量 | 月均点击热度 | string（带w）或 int | 0 |
| 4 | **下拉词数量** | 关联的下拉词个数 | int（无单位） | `-` |
| 5 | 字数 | 关键词字符数 | int | 未填 |
| 6 | 竞争程度 | 平台内竞争激烈程度 | **柱状图（div）** + 数值 | 柱状图空 |
| 7 | 推荐理由 | 平台自动打的标签 | 字符串，多个标签空格分隔 | `-` |
| 8 | 操作 | 行级操作 | 链接/按钮 | — |

- 每列表头右上角都有上下箭头，支持点列排序
- "核心指标"区是另一组筛选入口（月均搜索量、月均点击量、字数、竞争程度、推荐理由，每个右侧带筛选图标）

## 八、表格行内字段详解

### 8.1 关键词（标题列）

- 文本字符串，可能包含中文、英文、数字、特殊符号（`丨` `+` `（）` `?` `#` 等）
- **HTML 特殊处理**：搜索词 tab 下，关键词中的某些子词会用 `<span>` 高亮标出命中片段（红色），如"美团外卖丨外卖**美食****奶茶****咖啡**水果"中"美食""奶茶""咖啡"是橙色高亮（命中搜索词"奶茶"）
- 关键词的命中部分用 HTML 标签切分，多个 `text` 节点 + `<br>` 拼接

### 8.2 月均搜索量

- **范围**：从 0（无数据）到 2.2亿+（`2258.75w`）
- **格式变体**：
  - 字符串带 `w`（万）：`2258.75w`、`1511.08w`、`170.28w`、`1.1w`、`8.6w`、`20.9w`
  - 纯整数：`1200`、`900`、`1800`、`4023`、`2300`、`10000`
  - 0：冷门词
  - `-`：无数据

### 8.3 月均点击量

- 同月均搜索量，格式与空值规则一致
- 通常比搜索量小 1-2 个数量级
- 案例：
  - 顺丰速运+：搜索量 2258.75w，点击量 170.28w
  - 餐饮奶茶：搜索量 22.76w，点击量 100

### 8.4 下拉词数量（仅搜索词 tab）

- 纯整数，无单位
- 范围：从 0（无下拉词）到 20.9w+（如"奶茶"本身下拉词数量是 20.9w）
- 案例：餐饮奶茶 35、奶茶小程序点单 434、美团外卖古茗奶茶 5、古茗奶茶 2675

### 8.5 字数

- 纯整数（字符数）
- 范围：1（极短词）到 17+（长尾标题词）
- 案例：奶茶 2、一点点 3、肯德基 3、滴滴出行 15（"滴滴出行丨打车骑行公交租车代驾"算 15 个字符）

### 8.6 竞争程度

- **展示形式**：柱状图（`<div>` 嵌套多个 `<i>` 元素，类似柱状统计图）+ 无具体数字
- 视觉上柱高反映竞争强度（柱越多越高代表竞争越激烈）
- 详情页摘要场景下为空：`-`

### 8.7 推荐理由

- 多标签字符串，多个标签用**空格**分隔
- 标签可重复出现，**0 ~ 4 个并存**
- 完整标签枚举（穷举覆盖 15+ 页面后确认）：

| 标签 | 含义（推测） | 出现频率 |
|---|---|---|
| `黑马词` | 近期热度快速上升 | 高 |
| `同行买词` | 竞品正在投放 | 高 |
| `高点击词` | 点击率高于平均 | 高 |
| `蓝海词` | 竞争小、流量尚可 | 中 |
| `飙升词` | 增速极快 | 低 |
| `高跑量词` | 实际带来跑量的词 | 中 |
| `-` | 无标签 | 中 |

- 组合示例：
  - `黑马词 同行买词 高点击词`
  - `黑马词 同行买词 高点击词 蓝海词`
  - `同行买词`
  - `-`

### 8.8 操作列

每行 5 个操作入口（链接/按钮）：

| 操作 | 含义 |
|---|---|
| 详情 | 跳转到该词的 `/WsoKeyWordDetail/detail?keyword=...` 详情页 |
| 标题 | 复制或编辑标题 |
| 收藏 | 加入收藏夹 |
| 做图 | 一键生成配图 |
| 视频 | 一键生成视频 |

## 十、付费墙机制（关键约束）

整个 KeywordWSO 板块有**两类付费墙**，影响爬虫策略：

### 10.1 列表付费墙（左侧）

- 触发位置：左侧分词/全部列表，滚动到底
- 文案：「权限不足，升级会员版本可查看更多数据~」
- 按钮：「购买会员」
- 影响：左侧列表只能看到前 ~50 个分词/词条

### 10.2 Tab 付费墙（行业词）

- 触发位置：点击「行业词」tab 立即触发
- 文案：「权限不足，升级会员版本可查看更多数据」
- 弹窗完整内容：

  > 旗舰版：适合中大型企业高效协同使用，全网性价比首选
  > 企业版：适合中小公司及工作室便捷协作
  > 您当前是 个人版会员，剩余340天

- 价格信息（页面中显示）：

## 十二、单位与数据格式约定（重要！）

> 这一节是爬虫数值解析的核心。

### 12.1 月均搜索量 / 月均点击量

| 格式 | 实际值 | 出现频率 |
|---|---|---|
| `2258.75w` | 22,587,500 | 极高 |
| `1.1w` | 11,000 | 高 |
| `8.6w` | 86,000 | 高 |
| `20.9w` | 209,000 | 中 |
| `4023` | 4,023 | 中（不到万） |
| `100` | 100 | 中 |
| `0` | 0 | 冷门 |
| `-` | 无数据 | 边界 |

**解析规则**：
- 含 `w` 后缀：去掉 `w` 乘以 10000（注意保留小数）
- 纯数字：原样使用
- 遇到 `0` 或 `-` 或 `null`：标记为缺失

### 12.2 下拉词数量 / 字数

- 始终是**纯整数**，无单位
- 范围：0 到 20w+（下拉词数量）、1 到 20+（字数）

### 12.3 竞争程度

- DOM 表现为柱状图，无法直接读到数字
- 解析时建议抓取**柱状图中 `<i>` 元素的数量**作为竞争强度近似值
- 详情页摘要场景下可能是 `-`

### 12.4 推荐理由

- 字符串，多标签空格分隔
- 解析时按空格 split 后过滤掉 `-`

## 十七、爬虫策略建议

1. **API 拦截 vs 页面解析**：优先用 Playwright 拦截 XHR/fetch 响应，参考 `aidso_keyword_heat_service.py` 的 `handle_response` 写法。新增 `competitive_word` 和 `competitiveTop` 两条匹配规则。

2. **字段统一**：进入数据库前对单位做归一化：
   - `xxxw` → `int(float(xxx) * 10000)`
   - `yyy` → `int(yyy)`
   - `0` / `-` / `None` → `NULL`

3. **推荐理由**：字符串 `split(' ')` 后取 list 存入数据库，标量用 `,` 拼接作为展示。

4. **付费墙处理**：行业词数据抓不到就用 placeholder（如 `industry_locked: true`）显式标记，不要伪装成"无数据"。

5. **批量抓取优化**：当面对大量关键词（如本项目 100+ 个港险词），用单个模式 + 关键词循环；分词/全部切换是人工探索用的，不应进入自动抓取流程。

6. **登录态复用**：详情页和搜索页共用同一登录态，token 存在 `localStorage` 的 `token` key。Playwright 持久化 profile 已能复用。

7. **左半分词树 vs 右侧表格**：左侧是**导航**，右侧是**数据**。爬虫只关心右侧表格的 XHR 响应，左侧分词树不需要解析。
