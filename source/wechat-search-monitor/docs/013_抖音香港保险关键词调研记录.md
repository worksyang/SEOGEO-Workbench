#### 第五批的核心反常识发现

1. **"香港储蓄险"是港险抖音生态第二大品类词**（3.24w），仅次于"香港保险"(4.31w)。这说明抖音港险用户**最先想到的细分是储蓄险**，不是"重疾险/年金险"。

2. **"港险"作为简称词（1.88w）流量接近"香港保险"**。说明抖音用户更倾向用简称。

4. **"香港保险经纪人"系列是真的大词**——经纪人视角是抖音港险生态的隐藏主战场，比某些单品词的流量都高。

## 搜索量"高/低"标准（动态调整）

基于第1-5批数据，目前对抖音港险词的搜索量分布有了一个初步判断：

| 等级 | 月覆盖范围 | 词数估计 | 示例 |
|---|---|---|---|
| 巨型 | > 1w | 极少 | 香港保险 4.31w、香港储蓄险 3.24w |
| 大型 | 5000-1w | 少量 | 香港储蓄险买后真实收益 2.40w |
| 中型 | 1000-5000 | 部分 | 香港储蓄险推荐 1888、香港安盛 1591 |
| 小型 | 200-1000 | 较多 | 香港分红险 633、宏利宏挚传承 427 |
| 微流量 | < 200 | 大量 | 大部分单品词、保单功能词 |
| 无流量 | 0 | 部分 | 一些专业词、刚出的新产品 |

**当前判断标准**：
- 月覆盖 **> 1000** → 值得做内容（中型以上）
- 月覆盖 **200-1000** → 可作为长尾覆盖（小型）
- 月覆盖 **< 200** → 暂不投入
- 同时**有"商机词"标签或子词被打"商机词"** → 优先级提升一档

**这个标准会在后续跑词过程中继续校准。**

## 调研方法论沉淀（2026-06-11 复盘）

> 这一节是这次跑 39 个词后沉淀出来的**通用方法论**，下一次跑词应该直接按这个 SOP 走。

### 一、跑词顺序：先广后窄，先找桶再捞鱼

**错误示范（这次早期犯的）**：
按预设 100 词表逐个验证 → 大量词是 0 或 pending → 浪费 60% 的时间

**正确顺序**：

```
Step 1：先用"简称"试探 1-2 个大词（港险 / 香港保险 / 香港储蓄险 / 香港重疾险）
        目的：快速定位品类流量天花板
        输出：知道这个品类的"主词流量档"是 1w+、5k+、还是 1k+

Step 2：跑主词的全部子词（搜索词 tab 前 20 行 + 左侧分词树）
        目的：拿到最浓缩的"心智词云"
        输出：野生词清单 + 决策路径初步猜测

Step 3：从野生词里挑 5-10 个高月覆盖词去 DSO 验证
        目的：确认哪些野生词有真流量，哪些是平台自动生成的"长尾假词"
        输出：高价值词候选池

Step 4：从候选池里挑 1-2 个做"全维度解剖"
        目的：看 5 个 tab + 左侧分词树 + 7 日数据
        输出：1 个完整的心智路径案例
```

**关键原则**：**不要从"产品词"开始**（太窄），**从"简称/品类词"开始**（最宽）。

### 二、5 个数据源的优先级

| 数据源 | 价值 | 限制 |
|---|---|---|
| 搜索词 tab（主表） | ⭐⭐⭐⭐ 主词+核心子词 | 个人版只显示 20 行 |
| 左侧分词树 | ⭐⭐⭐⭐⭐ **最浓缩的词云** | 50+ 词后触发付费墙 |
| 下拉词 tab | ⭐⭐⭐ 数据=搜索词 tab | 字段更少，无新增价值 |
| 相关词 tab（DSO）| ⭐ DSO 宽泛无意义 | 与 WSO 含义不同，不能混用 |
| 行业词 tab | 🚫 个人版付费墙 | 完全看不到 |

**核心结论**：**优先看左侧分词树**，那 50 个分词比搜索词 tab 的 20 行还浓缩。

### 三、5 类"接近成交"价值信号

按价值从高到低：

| 信号 | 含义 | 例子 |
|---|---|---|
| **被打"商机词"标签** | AIDSO 判定用户有需求 | 香港储蓄险测评、收益如何转入内地 |
| **"买后/购买条件/如何买"** 字面 | 决策后期 | 香港储蓄险买后真实收益、购买条件 |
| **"靠谱吗/真实情况/真实收益"** | 决策中期（信任验证） | 香港储蓄险靠谱吗、香港友邦保险靠谱吗 |
| **"测评/推荐/排行/对比"** | 决策前期 | 香港储蓄险测评、香港保险排名前十名 |
| **高 7日均值（>100）+ 中高月覆盖** | 真实活跃需求（不是事件泡沫） | 香港储蓄险买后真实收益 7日 310 |

**反常识发现**：**"买后真实收益"比"测评"更接近成交**——用户搜"测评"是还没买；用户搜"买后真实收益"是已经买了或正在决策要买，**且对结果还有疑问**（高意图 + 高质疑）。

### 四、4 种"空态"的区分

| 空态 | 平台表现 | 业务含义 | 处理策略 |
|---|---|---|---|
| 完全空 | 0 results | 平台未收录 | 标记"未收录"，可放弃 |
| pending_collect | 0 + "点击收录" | 平台有索引但未完成数据采集 | **等几天再回来看**，可能是新词 |
| 主词有/子词全 0 | 主词 100+、子词 0 | 词刚出现 | 子词可观察 |
| 主词 0/子词有 | 反向情况 | 平台抓取逻辑问题 | 跳过 |

**反常识**：DSO 的"暂无数据，系统将为您收录"和"点击收录 数据更新中"是**两种不同的空态**。前者是词不存在，后者是词存在但平台还没统计完。

### 五、抖音 vs 微信的关键词生态差异（重要！）

| 维度 | 抖音 | 微信 |
|---|---|---|
| 命名习惯 | 简称（港险、保单） | 全称（香港保险） |
| 决策路径 | 测评、买后真实收益、靠谱吗 | 区别、怎么买、值不值得 |
| 流量集中度 | 极度长尾，少数大词 + 大量小词 | 更分散 |
| 经纪人视角 | 强（佣金/提成/返佣/经纪人/收入） | 弱 |
| 退保视角 | 强（退保/拿回/返佣后处理） | 弱 |
| IP 驱动 | 强（姚凯飞/凯哥/保险佬） | 几乎无 |
| 看后搜 | 重度影响所有数据 | 几乎无 |

**核心原则**：**微信视角的 100 词表不能直接平移到抖音**。抖音需要自己的分组逻辑（按心智层级，不是按产品类别）。

### 六、7 日趋势 = "真需求 vs 事件泡沫"的金标准

**这个还没完全验证但很关键**：
- 平稳曲线 = 真实稳定需求（值得做内容）
- 尖峰 = 事件驱动（短期流量，可能已过气）
- 7日均值 × 30 远小于 月覆盖 = 词曾火已衰减
- 7日均值 × 30 远大于 月覆盖 / 30 = 词在升温期

**操作方法**：hover 折线图 7 个 marker 拿到 7 日精确数据（010 文档第 24 章方法）。**这是判断"看后搜"的最有效手段**。

### 七、跑词效率优化

1. **批量跑**：每个 keyword 跑一次 5-6 秒，10 个词一批共 50-60 秒。**不要逐个跑**。
2. **首选 chrome-devtools**：evaluate_script 直接读 DOM，比 playwright 的 table reading 稳定。
3. **优先用 navigate_page(url=...)**：避免每次填搜索框的延迟。
4. **个人版 20 行限制**：单 tab 看不到 100+ 的完整数据。**靠分词树补足**。
5. **不要展开"超级导出"**：那是另一个付费功能。

### 八、判断"高价值"的四维标准（综合）

一个词是否值得做内容，应该看：

| 维度 | 权重 | 怎么判断 |
|---|---|---|
| 月覆盖（流量） | 30% | 至少 > 200，最好 > 1000 |
| 7 日均值（活跃度） | 20% | 至少 > 5，最好 > 50 |
| 商机词标签（接近成交） | 30% | 被打商机词标签，或子词被打商机词 |
| 子词丰富度（决策路径深度）| 20% | 下拉词数量 > 10，且子词有差异化 |

**综合评分**（自评）：> 70 = T1、50-70 = T2、30-50 = T3、< 30 = 放弃
