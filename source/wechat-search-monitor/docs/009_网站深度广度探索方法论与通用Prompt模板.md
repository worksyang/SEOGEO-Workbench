# 网站深度广度探索方法论与通用 Prompt 模板

> 本文档基于 2026-06-09 对 AIDSO 爱搜 KeywordWSO 板块的探索经验整理。
> 目标：把"我探索到一半就卡壳"变成"我一定能把整个网站拆得干干净净"——不论是豆包、YouTube、淘宝、还是企业级 B 端后台。
> 配套文档：`008_AIDSO爱搜KeywordWSO页面结构详解.md`（15K 字，覆盖 16+ 种页面状态）。

---

## 一、本次探索的真实复盘

把这次对话剥到底层，其实发生了三件事：

| 阶段 | 现象 | 根因 | 教训 |
|---|---|---|---|
| 第一次失败 | 用户让"全面探索 15 个页面"，我只看了 3 个就开始写文档 | 我把"读完 3 个详情页"等同于"理解整个网站"，但实际还有 4 个主 tab、2 个左侧视图、2 个搜索模式、3 个会员等级等待验证 | 探索页面数 ≠ 探索完整度。一个网站有 N 个独立可点击入口，每一个都要点开 |
| 第二次失败 | 用户截了一张图，圈出"分词""全部""4 个 tab"等关键控件，我才意识到左侧还有一整块我完全没碰 | 我用 `get_clickable_elements` 只看了首屏，没有滚动、没有用 `evaluate` 强制看全 DOM | 工具默认输出 ≠ 页面全貌。`innerText` 是被截断的，`clickable elements` 是首屏的，**真正的全貌要靠主动 `scroll` + 主动 `evaluate` 关键结构** |
| 第三次成功 | 重新规划成"4 tab × 2 视图 × 5 关键词 × 2 模式"的矩阵，逐项点击，穷举 16+ 状态，最终写出了 1.5 万字 | 找到了正确的"页面状态空间分解"思维——把网站拆成"维度 × 维度 × 维度"的笛卡尔积 | **探索 = 维度枚举**，不是"点几个看起来重要的页" |

一句话总结：**"全面"是穷举出来的，不是感觉出来的。**

## 二、根因诊断：为什么会"3 页就卡壳"

我把失败模式抽象成 4 个可命名的反模式，列出来供以后对照：

### 2.1 反模式 A：把"读完一个详情页"当作"理解整个网站"

错例：打开 `?keyword=友邦` → 看详情 → 写文档。

真相：友邦只是"搜索词" tab 的一行，详情页里还有"相关词""相关词TOP100"两个 tab；搜索结果页还有"下拉词""行业词"两个 tab；"行业词"又触发了付费墙，要单独标记。一个详情页只是冰山一角。

### 2.2 反模式 B：用"主要功能"做探索清单

错例：只点了看起来"主要"的搜索和详情。

真相：付费墙、屏蔽词、列表配置、批量模式、文件上传、导出按钮——这些"次要"功能往往藏着真正的限制、字段枚举、空值场景。少了任何一个，最终文档都不完整。

### 2.3 反模式 C：信任"页面似乎没东西"

错例：点击"相关词"tab，看到 "暂无数据"，以为就完事了。

真相："暂无数据"本身就是关键事实——它说明这个关键词在 AIDSO 的相关词库里**没有被收录**，这是爬虫要标注的"空值类型"之一。要把所有"空"的状态都记下来：暂无数据 / 0 / `-` / 付费墙 / 列表超限截断 / 暂无该字段。

### 2.4 反模式 D：以为"看到了就够了"

错例：截图发现 "C柱状图" 觉得就是个图表，跳过。

真相：柱状图是"竞争程度"字段在 WSO 板块的**特定渲染方式**——没有数字，只有 `<div><i><i>...`，这是字段级事实。最终爬虫拿到的是 HTML 而不是数字，必须在文档里讲清楚。

## 三、成功的探索方法：六维矩阵

把网站想象成**一个高维状态空间**。每个可点击的控件是一个维度，控件的每个状态是一个取值。完整探索 = 遍历这个笛卡尔积。

对 KeywordWSO 而言，状态空间是：

```
关键词       × {友邦, 顺丰, 蜜雪冰城, 安盛盛利2, 奶茶, 伯牙}   = 6 维
左侧视图     × {分词, 全部}                                   = 2 维
主 tab       × {搜索词, 相关词, 下拉词, 行业词}              = 4 维
搜索模式     × {单个, 批量}                                   = 2 维
频道         × {全站找词, 抖DSO, 红RSO, 快KSO, 微WSO, 拼BSO} = 6 维
会员等级     × {个人版, 旗舰版, 企业版}                       = 3 维
```

完整状态数 = 6 × 2 × 4 × 2 × 6 × 3 = **1728** 种理论组合。

实操当然不需要 1728 次点击——但你要知道这个**天花板**，并且确保你的探索至少覆盖了**控制变量后的代表性样本**：

- 至少 3 个不同热度档位的关键词（超亿 / 千万 / 百 / 无数据）
- 至少 2 个左侧视图状态
- 全部 4 个主 tab
- 2 个搜索模式
- 至少 1 个付费墙触发场景
- 至少 2 个不同分词/长尾词点击场景

> **目标不是穷举 1728 种**，而是**保证每个"维度"的"边界"都被摸到**。最终代表性样本数 = 维度数 × 2-3 ≈ 15-30 个页面状态。

## 四、探索的标准 SOP（7 步）

把这套方法论固化成 7 步流程，每一步都用具体动作+产出物绑定：

### Step 1：枚举"页面维度"（5 分钟，纯思考）

打开首屏，回答以下 8 个问题，把答案列成清单：

1. 顶部有几个可点击的全局 tab / 频道？
2. 主要的输入控件是什么？有几种模式（单个/批量、模糊/精确）？
3. 右侧或下方有几个主 tab？
4. 左侧有侧栏吗？侧栏有几个视图切换？
5. 列表区域有没有分页器？分页器有几个状态（首页/末页/跳转）？
6. 列表上方的"筛选区"有哪些维度？每个维度有几个可选值？
7. 有没有会员/付费相关的弹窗或按钮？可能触发什么场景？
8. 底部有没有分页/导出/批量操作按钮？

产出物：**维度清单**（Markdown 列表，6-15 条）。

### Step 2：抽"代表性关键词/输入样本"（5 分钟，纯思考）

对每个**会接收输入**的控件，准备 3-5 个代表性样本，**覆盖边界**：

- 极值（最大、最小、零、负数）
- 长度（极短、极长、中等）
- 类型（纯数字、纯中文、中英混排、含特殊符号）
- 业务覆盖（热门/冷门/无数据/付费墙触发）

产出物：**输入样本清单**（每个输入控件配 3-5 个样本）。

### Step 3：枚举"点击路径"（10 分钟）

把首屏每个 `role=tab`、`button`、可点击 `<div>` 都点一遍，记录点击后发生什么：

- URL 变化？/ 弹窗出现？/ 区域刷新？/ 跳新页面？/ 付费墙拦截？

产出物：**点击路径清单**（每条包含"起点 → 终点 → 中间状态"）。

### Step 4：组合探索（核心，30-60 分钟）

用**控制变量法**：

- 固定其他维度，只切一个维度
- 每次切换后**完整记录**新页面：URL、关键文案、表格列、数据样本（至少 5 行）、空值表现

产出物：**页面快照矩阵**（表格，每行一个"维度组合 → 关键发现"）。

### Step 5：穷举字段细节（20 分钟）

针对核心数据区（通常是列表/表格）：

- 每个字段用什么单位？字符串还是数字？空值是什么？
- 字段值是否随输入变化？（比如"竞争程度"柱状图会变高）
- 字段在哪些 tab 出现？哪些 tab 缺失？
- 字段是否需要付费才能看？（打赏星号/付费标记）

产出物：**字段字典**（每字段含：类型、单位、空值、出现 tab、付费状态、解析规则）。

### Step 6：识别所有限制（10 分钟）

主动触发：

- 输入超长字符串
- 输入特殊字符（`/` `\` `?` `#` `&` 单引号 双引号 表情）
- 输入会触发付费墙的关键词
- 多次快速点击同一按钮
- 关闭网络看错误提示
- 切换到低权限账号

产出物：**限制清单**（每条含：触发条件、表现、对爬虫的影响）。

### Step 7：撰写交付文档（30-60 分钟）

按统一模板（见第六节），把 1-6 步的产出物整合成一份能让他人"看了就能动手"的文档。

## 五、探索提示词模板（可直接复用）

下面这套 prompt 适用于任何"复杂网站深度广度探索"任务。可以**直接复制**到下次探索任务的开头：

```markdown
# 任务背景
[一段话说明要探索什么网站、为什么探索、最终交付物是什么]

# 目标网站
- URL: ...
- 大致业务: ...
- 我已知的信息: ...
- 核心使用场景: ...

# 探索目标（必须全部满足）
1. 覆盖网站全部"主要交互维度"（tabs / 视图 / 模式 / 会员等级）
2. 覆盖至少 3 个不同热度/类型/状态的输入样本
3. 触发并记录所有付费墙 / 限制 / 报错
4. 列出所有字段 + 单位 + 空值 + 出现位置
5. 穷举所有标签/枚举值
6. 最终交付一份 ≥ 5000 字的 Markdown 结构化文档，让任何 Agent 看完就能动手

# 探索方法（务必遵守）
## A. 状态空间分解（必做第一步）
打开首屏，列出所有可点击控件对应的"维度"和"取值"，构成维度矩阵。
例如：4 tab × 2 视图 × 5 关键词 × 2 模式 = 80 种状态。

## B. 控制变量探索
- 一次只切一个维度
- 每个新页面状态都要完整记录：URL、关键文案、表格列、数据样本（前 5 行）、空值
- 至少 15-30 个代表性页面状态

## C. 主动 DOM 探测（不要被"看到"骗了）
- 每个 tab 切换后，立刻用 `browser_evaluate` 拿完整 `innerText`
- 用 `browser_get_clickable_elements` 拿全可点击元素
- 对隐藏区域用 `scroll` 滚动后再看
- 对付费墙区域，记录"被拦截"和"原本应有的字段"

## D. 字段级穷举
- 每个核心字段都问：类型 / 单位 / 空值 / 边界 / 哪些 tab 出现
- 数值字段问：是否有单位字符（w/万/%）？如何解析？
- 标签字段问：完整枚举值？多选/单选？分隔符？

## E. 限制清单
- 主动触发付费墙、长度限制、特殊字符报错
- 全部记录到"限制清单"小节

# 必须穷举的输入样本（每个输入控件 3-5 个）
1. 极热门/超级热门词（搜索量过亿）
2. 中等热度词
3. 冷门词（无数据 / 0 数据）
4. 极短词（1-2 字）
5. 极长词（>10 字）
6. 含特殊符号的词

# 交付文档结构（必须按这个写）
1. 板块定位与 URL 形态
2. 页面整体骨架（ASCII 框图）
3. 顶部全局导航
4. 各主要区域详解
5. 字段字典（含单位 / 空值 / 解析规则）
6. 标签枚举（穷举）
7. 付费墙与限制清单
8. 空值场景汇总
9. 与现有爬虫/系统代码的差异
10. 探索过的页面快照清单
11. 已知未覆盖/留待确认事项

# 禁止的反模式
- 只看 3 个页面就开写
- 把"详情页"当作"全部页面"
- 信任默认工具输出，忽视隐藏区域
- 跳过付费墙、空值、报错
- 用"主要功能"做探索清单
- 文档少于 5000 字 / 字段不穷举 / 标签不穷举

# 工具使用提示
- browser_navigate: 切换 URL
- browser_click: 切 tab / 切视图 / 切模式
- browser_evaluate: 用 `() => document.body.innerText` 拿完整文本
- browser_get_clickable_elements: 拿全可点击元素
- browser_screenshot: 关键场景视觉留档
- browser_press_key: ESC 关闭弹窗
```

## 六、配套的"输出文档结构模板"

最终交付的 Markdown 文档，至少要包含 11 个章节（顺序建议）：

1. **板块定位与 URL 形态** — 让读者 1 分钟知道"这是什么网站、URL 怎么拼"
2. **页面整体骨架** — 一张 ASCII 图，让读者秒懂"页面从哪到哪"
3. **顶部全局导航条** — 每个 tab / 频道 / 切换器
4. **搜索/输入区域** — 控件 + 模式
5. **主数据区（核心）** — 4 个主 tab、每 tab 的列、数据样本
6. **字段字典（核心）** — 每个字段：类型 / 单位 / 空值 / 解析规则
7. **标签 / 枚举穷举** — 完整列出所有见过的值
8. **付费墙 / 限制清单** — 触发条件、表现、对爬虫的影响
9. **空值场景汇总** — 8-10 种空态的对照表
10. **与现有代码的差异** — 已实现 vs 实际包含，明确缺口
11. **探索过的页面快照清单** — 15+ 条代表性页面状态

可选附加章节：爬虫策略建议、术语对照表、已知未覆盖事项、API 推断。

## 七、不同网站类型的探索变体

不同网站的核心维度不一样，下面给出 3 种典型变体，**维度矩阵的设计思路**不变，只是维度名不同：

### 7.1 豆包（AI 对话产品）

|---|---|

## 八、字段枚举与数据格式的穷举法

字段级探索是**最容易漏的环节**。给你一份"字段穷举 8 问"：

| # | 问题 | 示例回答 |
|---|---|---|
| 1 | 这个字段是什么类型？（string / int / float / bool / array） | string |
| 2 | 字段值的单位是什么？（w=万 / 元 / % / 个） | w = 万 |
| 3 | 字段值的格式是什么？（带后缀 / 纯数字 / 范围 / 区间） | "2258.75w" 或 "4023" |
| 4 | 字段为空时怎么显示？（0 / - / 空字符串 / 不显示 / 隐藏列） | "-" |
| 5 | 字段值是否在多个 tab 出现？哪些 tab 没有？ | 仅搜索词 tab 有"下拉词数量" |
| 6 | 字段是否需要会员/付费才能看？ | 行业词全部字段都需付费 |
| 7 | 字段的最小/最大边界值是多少？ | 字段范围 0 - 2.2 亿+ |
| 8 | 字段在 UI 上是文字还是图表？（柱状图/进度条/雷达图/徽章） | 柱状图，柱数代表竞争强度 |

> 这 8 问是**任何字段都要走一遍的标准流程**。漏问一个，最终爬虫解析就会出 bug。

## 九、付费墙与限制的标记方法

探索过程中遇到的"看不到的数据"绝不能跳过，**必须显式标记**。建议用以下结构：

```markdown
### 限制 X：[限制名称]
- 触发条件：[什么操作/什么 tab/什么值]
- 表现：[页面提示 / 弹窗 / 静默截断]
- 文案：[完整复制弹窗文案]
- 涉及数据：[哪些字段/哪些 tab]
- 绕过方式：[已知是否存在绕过 / 不推荐 / 需高权限账号]
- 对爬虫的影响：[必须 X / 不能 X / 用占位符]
```

付费墙以外的限制也要列：长度限制、特殊字符报错、频繁操作限流、登录失效、CAPTCHA 等。

## 十、自检清单（写完文档后逐条勾选）

写完 Markdown 文档后，用这份清单自检：

- [ ] 文档字数 ≥ 5000 字
- [ ] 至少 15 个代表性页面状态被记录
- [ ] 每个主 tab 都有 5+ 行真实数据样本
- [ ] 所有核心字段都回答了"8 问"
- [ ] 标签 / 枚举值已经穷举（不是只列见过的几个）
- [ ] 至少 1 个付费墙场景被记录
- [ ] 至少 3 种空值场景被记录（0 / - / 暂无数据 / 截断）
- [ ] 包含 1 张以上的 ASCII 框图
- [ ] URL 形态完整列出
- [ ] 与现有代码的差异被显式标出
- [ ] 字段解析规则明确（"2258.75w" → 22587500 之类）
- [ ] 探索过程被记录（哪个维度、什么关键词、发现了什么）
- [ ] 已知未覆盖的事项被诚实列出

> 自检不通过 = 文档未完成，必须补完。

## 十一、失败模式速查表（以后卡壳时自检）

| 症状 | 原因 | 解药 |
|---|---|---|
| 写到一半不知道接下来写什么 | 维度矩阵没建 | 回到 Step 1 重新列维度 |
| 字段解释总是空泛 | 没问"8 问" | 强制每个字段走一遍 8 问 |
| 文档不到 5000 字 | 探索样本不够 | 至少再补 5 个页面状态 |
| 出现"以上"、"其他"、"等" | 在偷懒 | 强制穷举，列全所有见过的值 |
| 误把详情页当作全站 | 反模式 A | 强制列出全部 tab 清单 |
| 没看到付费墙 | 没切到受限 tab | 主动点击所有 tab 触发 |
| 文档结构和 008 完全不像 | 没用统一模板 | 套第六节的 11 章节结构 |
| 多次重复同样的字段 | 缺章节化 | 每个字段只出现一次，跨字段引用 |
| 文档读完还是不知道怎么爬 | 缺"差异清单" | 加一节"与现有代码的差异" |

## 十二、配套关系

```
008_AIDSO爱搜KeywordWSO页面结构详解.md
  └─ 一个具体网站的探索结果（"WHAT"）
009_网站深度广度探索方法论与通用Prompt模板.md  ← 本文档
  └─ 探索任何网站的方法论（"HOW"）
```

**008 是 009 的实例**，**009 是 008 的方法抽象**。下次探索新网站时：

1. 把 009 第三节的"维度矩阵"用上
2. 把 009 第五节的 Prompt 模板复制到任务开头
3. 把 009 第六节的 11 章节结构作为文档骨架
4. 把 009 第十节的自检清单作为收尾 checklist

## 十三、最后一句话

> **全面 = 维度枚举 × 代表性样本 × 字段穷举 × 显式标记限制**。
>
> 不靠"感觉已经够了"，靠"矩阵走完了，标签穷举了，空值都标了"。
>
> 任何"应该够了"的想法，都意味着你只探索了 3 个页面就开写了。
