                    main.py 与 article_workflow.py 本质区别流程图
═══════════════════════════════════════════════════════════════════════════════

┌─────────────────────────────────────────────────────────────────────────────┐
│                           main.py 工作流程                                  │
│                      【数据采集 + 基础报告生成】                             │
└─────────────────────────────────────────────────────────────────────────────┘

开始
  │
  ├─→ 登录 WeRSS API
  │
  ├─→ 用户选择模式
  │   ├─→ 模式1：更新所有公众号文章（调用更新API，等待爬虫完成）
  │   └─→ 模式2：直接读取现有数据（不更新，快速）
  │
  ├─→ 遍历所有公众号
  │   ├─→ 获取每个公众号的文章列表
  │   └─→ 统计信息（今日/一周/总计）
  │
  ├─→ 第一阶段：批量更新/读取
  │   └─→ 为每个公众号生成独立CSV文件（按公众号分类）
  │
  ├─→ 第二阶段：生成每日报告
  │   ├─→ 筛选最近7天的文章
  │   ├─→ 按日期分组（20250621.md, 20250622.md...）
  │   └─→ 生成Markdown表格报告（标题 | 公众号 | 发布时间）
  │
  ├─→ 第三阶段：生成总数据库
  │   └─→ 更新总CSV文件（articles.csv，包含所有文章）
  │
  └─→ 输出统计汇总
      ├─→ 各公众号文章统计
      ├─→ 总体统计（今日/一周/历史总数）
      └─→ 生成文件列表

输出结构：
  output/
    ├─ 公众号1.csv          （单个公众号的所有历史文章）
    ├─ 公众号2.csv
    ├─ ...
    ├─ articles.csv         （所有公众号的总数据库）
    ├─ 20250621.md          （按日期组织的每日报告）
    ├─ 20250622.md
    └─ ...

特点：
  • 数据采集为主，不做筛选
  • 按日期组织输出
  • 生成统计报告
  • 不进行AI分类
  • 不进行内容转换（只生成链接表格）


┌─────────────────────────────────────────────────────────────────────────────┐
│                      article_workflow.py 工作流程                           │
│                    【智能筛选 + AI分类 + 内容转换】                          │
└─────────────────────────────────────────────────────────────────────────────┘

开始
  │
  ├─→ 登录 WeRSS API
  │
  ├─→ 获取最近N天文章（默认15天）
  │   └─→ 遍历所有公众号，收集文章
  │
  ├─→ 去重过滤
  │   ├─→ 扫描现有MD文件（检查是否已处理）
  │   ├─→ 加载已拒绝文章记录（rejected_articles.csv）
  │   └─→ 使用相似度算法（thefuzz）过滤重复文章
  │
  ├─→ AI智能分类
  │   ├─→ 提取文章标题列表
  │   ├─→ 批量发送给AI分类器（AIClassifier）
  │   └─→ 获得分类结果（类型标签）
  │
  ├─→ 匹配与筛选
  │   ├─→ 将AI分类结果与原始文章匹配（模糊匹配标题）
  │   ├─→ 根据类型筛选
  │   │   ├─→ 允许的类型 → 接受
  │   │   └─→ 拒绝类型 → 拒绝
  │   └─→ 生成两个列表：接受文章 / 拒绝文章
  │
  ├─→ 处理接受的文章
  │   ├─→ 按类型分组（不按日期）
  │   ├─→ 调用URL转Markdown服务（SomeURL2MD）
  │   ├─→ 转换为Markdown文件
  │   ├─→ 检查已删除文章（标题为"未命名文章"）
  │   └─→ 保存到对应类型文件夹
  │
  ├─→ 处理拒绝的文章
  │   ├─→ 转换为Markdown（仅转换，不OCR）
  │   └─→ 保存到"排除流"文件夹
  │
  ├─→ 更新拒绝记录
  │   └─→ 保存到 rejected_articles.csv（用于下次去重）
  │
  └─→ 输出统计报告
      ├─→ 总计发现文章数
      ├─→ 已跳过文章数
      ├─→ 接受文章处理结果
      └─→ 拒绝文章归档结果

输出结构：
  output_md/
    ├─ 安盛盛利2/              （按类型组织的文件夹）
    │   ├─ 公众号名_标题1_时间戳.md
    │   └─ 公众号名_标题2_时间戳.md
    ├─ 太保鑫相伴/
    ├─ 香港储蓄险/
    ├─ z非热门产品/
    ├─ 其他/
    └─ 排除流/                 （被拒绝的文章）
        └─ 公众号名_标题_时间戳.md

特点：
  • 智能筛选为主，只处理新文章
  • 按类型组织输出（产品分类）
  • 使用AI进行内容分类
  • 进行内容转换（URL → Markdown）
  • 自动去重和拒绝记录管理

═══════════════════════════════════════════════════════════════════════════════
                              使用场景对比
═══════════════════════════════════════════════════════════════════════════════

main.py 适合：
  • 需要定期更新所有公众号文章数据
  • 需要生成每日文章汇总报告
  • 需要统计各公众号发布情况
  • 需要导出CSV数据进行数据分析
  • 不需要对文章内容进行筛选

article_workflow.py 适合：
  • 需要自动筛选高质量文章
  • 需要按产品类型分类归档
  • 需要将文章转换为Markdown格式
  • 需要避免重复处理相同文章
  • 需要记录和排除不合格文章
