# 搜索意图与高价值词根映射模型

页面类型：底层认知聚合页  
状态：正式页  
最后更新：2026-05-06

本页沉淀 SEO / GEO 选题里的底层判断：什么搜索词更接近成交、哪些词根值得优先写、微信搜一搜和 AI 问答型平台的表达差异如何转成选题。

它不是关键词表，也不是投放词库。具体词、排名、发布时间和账号状态以后应进入选题账本；本页只保留可复用的判断模型、成交味排序、内容承接方式和搜索型文章打法。

## 1. 核心结论

搜索词的价值不由搜索量决定，而由用户离成交有多近决定。

在香港储蓄险、保费融资、IUL 这类高客单价产品里，最有价值的不是“香港保险怎么买”这种泛词，而是用户已经拿到方案、锁定产品、进入对比或风控审查时问出的窄词。

典型高价值信号：

- 已出现具体产品名。
- 已出现提领、现金价值、保证回本、分红实现率、优惠、停售、融资利率等决策变量。
- 已出现两个产品或两个方案的对比。
- 已出现“靠不靠谱”“有没有坑”“能买吗”“适合养老提现吗”这类临门一脚判断。

## 2. 交易距离分层

| 层级 | 词根形态 | 用户状态 | 内容目标 |
| --- | --- | --- | --- |
| S 级 | 产品 + 提领 / 优惠 / 停售 / 融资条件 | 基本已进入方案或渠道选择 | 截流、拆方案、给最后判断 |
| A 级 | 产品 + 缺点 / 分红实现率 / 保证回本 / 现金价值 | 已锁产品，正在做风控 | 建立信任、解释边界、阻断误买 |
| A 级 | 产品 A + 产品 B | 二选一或多选一 | 横评、做取舍、导向更优方案 |
| B 级 | 单一产品名 | 了解产品，意图不纯 | 建立产品认知入口 |
| B 级 | 人群场景 + 方案 | 有需求但未定产品 | 教育、筛选、引导咨询 |
| C 级 | 品类词 / 入门词 | 早期学习 | 品牌占位、长期认知 |
| C 级 | 骗局 / 负面泛词 | 高警惕但未必有明确方案 | 破冰、建立专业感 |

默认优先写 S / A 级词。C 级词只有在品牌占位、账号权重或热点切入需要时才优先。

## 2.1 真实成交味排序

在高客单价金融保险领域，搜索量不是第一指标。距离交易越近，词越具体，转化率越高。

真实成交味从强到弱：

1. **产品 + 动作 / 优惠 / 停售**：如 `盛利2提领`、`盈聚天下2优惠`、`盛利2停售`。用户往往已经拿到方案，在做最后确认、找折扣或赶窗口。
2. **产品 + 风险 / 负面 / 细节**：如 `盛利2缺点`、`信守明天分红实现率`、`盛利2保证回本`。这是临门一脚的风控审查，最容易建立专家信任。
3. **产品 + 对比**：如 `盛利2 星河2`。用户已经进入二选一阶段，一针见血的横评最容易截流。
4. **单一产品名**：流量可能更大，但意图混杂，小白、同行和准客户都会搜。
5. **人群场景词**：如孩子教育金、养老提现，有需求但未锁产品，需要较长教育。
6. **负面怀疑词**：如港险骗局、保费融资是不是骗局，适合破冰和曝光，但离具体成交更远。
7. **品类词**：如香港储蓄险、保费融资，适合品牌占位，不应消耗太多主力资源。

## 3. 高价值词根

### 3.1 产品动作词

这类词最接近成交，因为用户通常已经在看计划书或被代理人推进到方案执行阶段。

- 提领 / 提取 / 养老提现
- 现金价值 / 退保价值
- 优惠 / 折扣 / 首年优惠
- 停售 / 限额 / 额度
- 年金转换 / 保单拆分 / 更换受保人
- 保费融资 / 融资利率 / 银行贷款

写法重点：用具体表格、计划书口径、现金流变化和风险边界回答，不要只做产品介绍。

### 3.2 产品风控词

这类词说明用户已经有购买倾向，但不放心。

- 缺点 / 坑 / 风险
- 分红实现率
- 保证回本
- 预期 IRR
- 长期收益靠不靠谱
- 能不能养老 / 能不能传承

写法重点：先承认焦虑，再拆边界。不要硬洗地，也不要用“分红不保证”这种行业共性废话替代产品级判断。

### 3.3 产品对比词

这类词通常是强成交词，因为用户已经有候选产品池。

- `盛利2 vs 星河尊享2`
- `环宇盈活 vs 盛利2`
- `宏挚传承 vs 匠心传承2`
- `信守明天 vs 盈聚天下2`
- `保费融资 vs 直接买储蓄险`
- `IUL vs 杠杆寿`

写法重点：不要平均介绍两个产品，而是围绕一个决策问题切：谁更适合养老提现、谁更适合传承、谁前期现金价值更舒服、谁对融资利率更敏感。

### 3.4 金额与方案词

金额、缴费期和方案词会显著提高成交味。

- 5 年缴 / 2 年缴 / 趸交
- 年缴 10 万 / 50 万 / 100 万
- 258 提领 / 557 提领
- 大额保单 / 家族传承 / 孩子教育金
- 年化 13% / 贷款利率 / 压力测试

写法重点：金额词不能孤立使用，必须和现金流、资金占用、回本期、贷款利率或分红假设一起解释。

## 4. 微信搜一搜与 GEO 的表达差异

### 4.1 微信搜一搜

微信搜一搜更像搜索框，用户表达短、硬、词组化。

适合标题形态：

- `安盛盛利2提领，到底能不能当养老金？`
- `保诚信守明天分红实现率，真正要看哪几项？`
- `富卫盈聚天下2优惠，值不值得为了折扣上车？`

核心要求：标题要命中词根，但正文必须真正解决问题。长期前三不是只靠标题，而是靠账号权重、完读率和问题解决率。

### 4.2 GEO / AI 问答平台

豆包、Kimi、DeepSeek、元宝更像私人顾问，用户会问完整问题。

适合问题形态：

- `安盛盛利2的提领真的适合养老吗？`
- `友邦环宇盈活和盛利2谁更值得买？`
- `保费融资年化13%到底是不是骗局？`
- `IUL和杠杆寿给孩子留钱哪个更稳？`

核心要求：保留自然问句、焦虑和场景，不要压缩成干巴巴关键词。

## 4.3 两套内容分开做

微信搜一搜和 GEO 的内容形态必须分开。

- **微信搜一搜**：标题打“核心名词 + 修饰词”，正文解决一个窄问题。
- **GEO / AI 问答平台**：标题和正文保留完整长句、情绪痛点和顾问式解释。

资源应严重倾斜于“小而准的交易词”。大词如“香港储蓄险”适合做账号权重和品牌占位；小词如“盛利2保证回本18年合理吗”虽然搜索量低，但用户手里更可能已经握着方案和预算。

## 5. 选题生成规则

每个高价值搜索词进入创作前，要先问四个问题：

1. 用户是否已经锁定产品或方案？
2. 用户到底卡在收益、风险、对比、执行还是渠道优惠？
3. 这篇内容能否一次性解决他的核心疑虑？
4. 这个疑虑应回填到产品母页、底层认知、创作框架，还是只进入选题账本？

如果一个词只能带来泛流量，但不能反推出用户疑虑或成交节点，就不应优先进入 Wiki。

## 5.1 关键词价值指标

判断一个搜索词值不值得写，按以下顺序：

1. **是否有真实客户问过**：私域里有 2 个以上客户问过同一个具体问题，就可以视为黄金词。
2. **是否能承接咨询**：词背后能否自然引出测算表、计划书对比、避坑清单或一对一筛选。
3. **是否绑定具体产品 / 优惠 / 方案**：越能落到产品、缴费期、提领、现金价值、融资利率，越接近交易。
4. **当前搜索结果质量是否差**：如果前三名都是营销废话或陈旧信息，就有降维打击机会。

搜索量和竞争强度只作为次要参考，不能压过交易距离。

## 6. 内容承接方式

### 6.1 一文主打一件事

搜索型文章默认只主打一个核心意图，最多顺带 2 到 3 个相关元素。

例：主打 `盛利2提领`，可以顺带解释 `258`、`现金价值`、`养老现金流`；不要同时塞入停售、优惠、星河2对比、分红实现率、宏观美元配置。

### 6.2 小词要做深，不要做宽

小词的价值来自精准。用户问到 `信守明天分红实现率`，他要的不是保诚品牌故事，而是：

- 这个产品目前有没有长期实现率数据。
- 老产品实现率争议能否外推到新产品。
- 看分红实现率时还要看哪些口径。
- 计划书里的 IRR 哪部分是保证、哪部分是假设。

### 6.3 负面词要拆边界

负面词不能直接洗白。

例：`保费融资是不是骗局` 的正确拆法是：

- 不是简单骗局，而是杠杆结构。
- 收益来自保单预期回报高于贷款成本。
- 风险来自贷款利率、续贷、分红兑现、汇率和现金流纪律。
- 不适合短期周转、低风险偏好和只想保本保收益的人。

### 6.4 一鱼多吃

搜索词是需求切片，不应该只服务一篇公众号文章。

高价值搜索词至少可以转成四类资产：

1. **产品测评页 / 横评页**：承接长尾精准流量。
2. **GEO 问答答案**：把短词改写为自然问句，供 AI 问答平台抓取和复述。
3. **私域话术 / SOP**：把高频疑虑转成顾问解答话术。
4. **短视频脚本**：把交易意图词放到前三秒钩子里。

## 6.5 当前产品池提示

后续选题账本可以继续验证，但当前写作系统应重点关注三类产品池：

- **已验证强势覆盖**：安盛盛利2、友邦环宇盈活、香港储蓄险泛词。
- **亟需补齐的空白区**：宏利宏挚传承、保诚信守明天。
- **高净值复杂结构**：保费融资、IUL、杠杆寿。

这类信息属于阶段性策略，进入正式写稿前仍要结合最新搜索结果、账号权重和产品状态复核。

## 7. 入库边界

- 具体关键词、排名、发布时间、账号状态：未来进入 `wiki/来源账本/选题账本/`。
- 搜索意图、词根、交易距离：保留在本页。
- 产品高频疑虑：回填到对应产品母页，但不堆关键词。
- 跨产品高频漏洞：回填到 `[[高价值缺陷与漏洞]]`。
- 稳定写作结构：回填到 `wiki/创作框架/` 或候选区。
