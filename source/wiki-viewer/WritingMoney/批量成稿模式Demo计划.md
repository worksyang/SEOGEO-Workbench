# WritingMoney 批量成稿模式 Demo 计划

## 目标

WritingMoney 从单一的“新建文章工作台”升级为两种模式：

- **母文章铸造**：现有页面保持原样，用于慢速、高质量、可复用母文章生产。
- **批量成稿**：新增模式，用已有母文章、关键词队列和引流钩子快速生成多篇成稿。

核心判断：两种模式不应该拆成两个系统，也不应该共用同一个重流程页面。它们应该共享外壳、项目列表、抽屉、三步进度和视觉风格，但第一步工作区完全不同。

## 模式一：母文章铸造

保持现有 Demo 形态，不改样式和主流程。

三步：

1. 写作决策：素材、URL 临时素材、模板推荐、AI 初版方案。
2. 写作包：把决策固化为可交给 Agent 的完整上下文。
3. 成稿：输出到 output 和工作目录。

定位：贵模型、低频、人深度介入、沉淀母文章资产。

## 模式二：批量成稿

新增模式，重点是速度和队列感。

三步：

1. 批量配置：选择母文章、查看关键词队列、匹配引流钩子、设定生成策略。
2. 批量包：把母文章、关键词、钩子、生成规则整理成批量输入包。
3. 成稿队列：展示多篇成稿的运行、完成、返工状态。

定位：中等模型、高频、半自动或全自动、服务每日多账号发布。

## 批量配置页应包含

- 当前母文章卡片：标题、版本、质量状态、关联主题集群、可回到母文章铸造返工。
- 关键词队列：多条关键词，每条展示意图、目标账号层、推荐钩子、状态。
- 引流钩子推荐：由 LLM 自动匹配，用户可切换，例如计划书解读、提领压力测试、资料领取、优惠折扣。
- 批量策略：每关键词生成篇数、结构差异、事实校验、重复度检查、发布队列。
- 运行概览：已完成、运行中、需要返工、等待生成。

## 返工逻辑

如果批量成稿发现“母文章本身有问题”，不在批量模式里展开素材和模板。批量模式只提供一个清晰入口：**回到母文章铸造返工**。

Demo 行为：

- 点击返工入口后切到母文章铸造模式。
- 当前阶段回到写作决策。
- 页面提示“已回到母文章铸造，可修母文章底板”。

## UI 原则

- 保持 Top 三项目风格：白底、细边框、紧凑、蓝色轻强调。
- 切换模式要轻，不做大 landing。
- 批量模式更密、更像队列，不展示素材长表和图片抽屉。
- 母文章铸造现有样式不改。
- 解释性文字尽量短，必要说明放 tooltip。

## 静态 Demo 验收标准

- 顶部或主区有清晰模式切换：母文章铸造 / 批量成稿。
- 母文章铸造切回后，原有写作决策页面视觉和功能基本不变。
- 批量成稿显示专属三步：批量配置 / 批量包 / 成稿队列。
- 批量配置页能看到母文章、关键词队列、引流钩子、策略和运行概览。
- 点击“生成批量包”进入批量包；点击“模拟开始批量成稿”进入成稿队列。
- 批量成稿里有“回到母文章铸造返工”入口，并能切回模式一。
- `node --check WritingMoney/app.js` 通过。
- 浏览器打开 `http://localhost:18923/WritingMoney/index.html` 后无控制台错误，核心交互可点击。
