写作流程图：LLM 模式 vs Agent 模式

左：单 Prompt 直出 · 右：路由 + 多重装配 + 双闸门
LLM 写作模式（原流程）
一个大 Prompt 驱动模型直出全文
1
Markdown 知识库文件夹
一个有结构的文件夹，存放大量 .md 原文，每篇可配同名 .txt；全库有兜底 SEO.txt。
📁 output_md/ ├─ 微信搜索结果/ │ ├─ 保诚.md 保诚.txt │ └─ 万通.md （无同名 txt） ├─ 热门产品/ │ └─ 友邦盈御3.md └─ SEO.txt ← 兜底关键词库（一行一个）
2
遍历读取所有 Markdown
逐篇读取 .md 作为原文素材；原文已穿插图片链接并附带 OCR 信息。
3
匹配 Title（标题字段）
优先读同名 .txt 取标题；无则回退 SEO.txt 提取关键词。
4
载入写作 Prompt 模板
载入总 Prompt（形如 prompt0909.md），填入 {title}、当前时间等字段。
▸ Prompt 核心逻辑（超简略）
  1. 人设：懂行的闺蜜/兄弟，不是老师科普。
  2. 风格：口语化、沉稳，激发信任感。
  3. 改写：基于原文重构，揭示产品缺陷与"圈里人"风险。
  4. 产品库：内置 12 款热销港险优劣势速查表。
  5. 图文：保留原图链接 + 按 OCR 插图。
  6. 合规：保司/产品名按规则替换，删引流信息。
  7. 输出：Markdown 代码窗口，含 SEO 关键词，~6000 字。
5
LLM 生成图文并茂的新文章
Prompt 直接驱动模型写出整篇正文，保留原图链接、按 OCR 插图，OCR 注释文字不进正文。
6
关键词替换合规流程
对成稿做关键词替换：保司/产品名按规则替换、删除引流与无关信息。
7
合规校验通过
对替换后的成稿做最终校验，确认无违规内容。
8
输出可发布的 Markdown 成稿
一篇 Markdown 文章，图文并茂，可直接发微信公众号。
Agent 写作模式（新流程）
路由 + 框架/积木/事实来源三重装配 + 破窗前置 + 红线后置
1
知识库文件夹（Raw 层 + Wiki 层）
比 LLM 模式多了 wiki/ 编译层：Raw 原文 + 同名 txt 兜底 SEO.txt，之上是分门别类的写作弹药。
📁 output_md/ ├─ 微信搜索结果/ ← Raw 原文层 │ ├─ 保诚.md 保诚.txt │ └─ 万通.md ├─ SEO.txt ← 兜底关键词库 └─ wiki/ ← 编译层（写作弹药） ├─ 产品母页/ 保诚/万通/友邦… ├─ 保司母页/ 友邦/保诚/宏利… ├─ 底层认知/ 分红/提领/税务边界 ├─ 创作框架/ 7 个写作框架+积木库 ├── 案例故事/ └─ 知识库规则/ 01-Agent创作执行流程
2
接收系统注入（身份 + 规则 + 上下文）
云端自动注入：写作身份（希希）、AGENTS.md 规则、环境信息、技能列表。Agent 无需主动"载入"，这些是每轮对话的固定上下文底色。
3
先读破窗机制（定终点）
写作前先理解结尾转化目标，带着终点写全文。结尾三步走：
打破"我懂了"幻觉 低门槛邀请"问一下不亏" 话不说完 · 不收口
▸ 这是 Agent 模式前置的"终点闸门"，LLM 模式没有
4
判断关键词 & 对象粗分
先判断题目命中的业务词，做对象粗分，再进入创作框架：
A 单一产品 B 单一保司/银行 C 多产品/横评 D 概念/机制/数据 E 趋势/流程/偶发
LLM 闸门 A：路由判断（三层拆解，非单一 LLM 调用）
A1 框架路由（规则优先，LLM 只处理灰区）
先跑 01 第4节关键词路由表做确定性匹配 → 唯一命中直接定框架；多个命中 → LLM 仲裁（输入仅：题目 + 候选框架一句话描述 ≈ 300字）
A2 积木选择（LLM，上下文极轻）
输入：题目 + 选定框架 + 10 个积木的一句话适用场景（从 01 第5节提取 ≈ 500字）→ 输出选 1-2 个积木
A3 事实来源（纯规则，不需要 LLM）
按 01 第6节对象→路径映射，用文件名 Glob 匹配 → 输出要读的文件列表
解法核心：闸门 LLM 只看路由表/一句话描述（≈ 1KB），不读全文。Agent 拿到决策后自己读全文 → 不破坏按需读取。不需要额外维护 summary，01 本身就是所有框架的 summary。
◆ 上下文装配阶段（Agent 核心，区别于单 Prompt）
5
路由：选 1 个主创作框架
按业务词命中选一个最贴近的框架，不混用多个：
保费融资 分红实现率 产品测评 产品横评 保司评价 条款黑盒 通用问题
6
选 1-2 个心理认知积木
只在思考中调用，正文不暴露积木名称：
品牌塌陷 静态幻觉 从众阻断 对比效应 锚定效应 信息黑盒 损失规避 确定性偏差 智力权衡透明 破窗与闭环剥夺
7
选事实来源（不全库扫描）
按对象读取对应来源，按需补充：
产品母页 保司母页 底层认知 案例故事 横评页 用户指定 Raw
LLM 闸门 B：事实充足检查
已读取文件列表 + 用户题目 → 固定 LLM prompt → 输出 {是否缺关键事实, 缺什么, 能否开始写}。不足则硬停，避免弹药不够时硬写导致编造数据。
8
套传播包装层
只处理打开率和第一屏，不替代主框架、不改事实：
标题含关键词+冲突感 开头120字给"别处看不到"的东西 首屏先冲突后解释
9
按框架写作正文
框架定推进顺序，母页「写作抓手层」定怎么翻译术语、先抓什么误会；图文并茂保留原图 + 按 OCR 插图。
LLM 闸门 C：框架合规校验
正文 + 选定框架名 → 固定 LLM prompt → 输出 {是否遵循推进顺序, 是否暴露内部结构, 积木是否只在思考中}。在红线检查前先做框架级自检，比通用红线更精准。
10
统一输出要求处理
图片弹药库挑选配图、品牌署名统一为保智库/希希、数据可追溯、排版加粗重点、命名 YYMMDD_标题.md。
11
红线检查（10 条）
客群锚定 / 事实边界 / 数据红线 / 语气边界 / 传播边界 / 转化方式 / 图片来源 / 框架隐形 / 读取边界 / 搜索词禁露。
▸ 这是 Agent 模式后置的"发布闸门"，LLM 模式只有简单合规替换
LLM 闸门 D：红线逐条结构化
10 条红线拆成固定 LLM prompt → 逐条输出 {红线项, 通过/不通过, 具体问题}。从 "Agent 自觉自检"变成"逐条强制校验"，避免自己查自己漏判。
12
输出 output/ 成稿
一篇可直接发布的 Markdown 文章，不残留占位内容、不暴露内部结构。
