## 1. 紧急安全事件：冻结代码字节码缓存被改写

### 1.1 事实

- 2026-07-16 04:38 左右，初版 `workbench/tests/wechat_write_replica.py` 及前置探针把
  `data/migration/wechat/freeze_20260716T024524+0800/payload/code-snapshot`
  直接加入 `sys.path`，随后 import 旧系统 `app.*` 模块。
- Python import 因未禁止 bytecode 回写，重写了冻结 `code-snapshot` 内多处
  `__pycache__/*.pyc`，主代理观察到相关文件 `mtime` 集中变化到
  `2026-07-16 04:38:32`。
- 这是**冻结永久只读边界违规**。即使 `.pyc` 内容 hash 可能不变，mtime 被改变也已经构成违规。
- 未对这些文件执行删除、恢复、touch、mtime 回拨或覆盖；保留现场供主代理审计。
- 本任务没有对旧真实源
  `/Users/works14/.claude/监控/wechat-ybxhyyh-top3`
  执行读取副本、写入、Git 操作或服务启动；安全复核时 8765 无监听，未发现旧源
  `run.py`、runner、watchdog 或本任务 pytest 遗留进程。

### 1.2 触发命令类型与影响边界

- 触发类型：
  - 以冻结 `code-snapshot` 为 `PYTHONPATH` 的 `python3` 只读探针；
  - 初版双副本 pytest harness 对冻结路径执行 `sys.path.insert(...)` 后 import。
- 已知影响：冻结 `code-snapshot/**/__pycache__/*.pyc` 的内容或元数据被 Python
  import 机制重写，至少 mtime 发生变化。
- 未知项：本轮不擅自恢复现场；最终受影响文件清单、原始 hash/stat 证据以主代理事件
  manifest 为准。

### 1.3 已采取的纠正

1. 立即停止双副本测试，确认无 pytest/Hub 子进程、无 8765 监听。
2. A 的代码改为先把冻结 `code-snapshot/app` **复制到 `tmp_path`**，忽略
   `.git`、`__pycache__`、`.pyc/.pyo`、`.DS_Store`，只从临时代码副本 import。
3. A 的 `app.db` 只从冻结包复制到临时目录，所有写入只落在临时 DB。
4. import 前设置 `sys.dont_write_bytecode=True`；临时代码副本也必须不存在
   `__pycache__` 和 `.pyc`。
5. A 的运行配置全部指向临时目录，不再把 `PROJECT_ROOT`、`NORMALIZED_DIR` 或
   `MONITOR_DATA_FILE` 指向冻结包。
6. 增加冻结源前后 manifest：对所有可能被复制/import 的
   `code-snapshot/app/**` 文件及冻结 `app.db` 比较文件集合、`size`、`mtime_ns`
   和 SHA-256；任何新增、删除或变化立即失败。
7. 增加硬路径门禁：冻结来源必须位于指定 freeze payload；旧真实源路径出现于
   来源、目标、`sys.path` 或已加载旧模块路径时立即失败。

### 1.4 安全事件结论

**事件状态：已停止继续破坏；修复后的隔离安全专项已 PASS。**
专项确认 A 从临时代码副本加载、临时副本无 `__pycache__/*.pyc`，且本次运行前后
冻结 `code-snapshot/app/**` 与冻结 `app.db` 的文件集合、size、mtime_ns、SHA-256
完全一致。该 PASS 只证明修复后未继续改变冻结源，不消除 04:38 已发生的违规。
本事件不标记为“无影响”，也不以内容 hash 可能相同为理由降级。

## 2. 双副本方法

- A：冻结 `app.db` + 冻结 `code-snapshot/app` 的临时隔离副本；使用进程内 Flask
  test client，不监听端口、不启动 scheduler、不触发 rebuild、抓取或 Aidso。
- B：临时 Hub SQLite；把冻结 registry 映射到 v3.3 核心/运行表，并建立
  `bootstrap/full/keyword/keyword_manage` 兼容投影。
- 对比项：HTTP status、JSON/HTML body、关键词与分组落盘状态、管理/主列表/详情
  投影、动态字段保留、重启持久化、命令、审计、双写回执、显式幂等、同键异输入
  409、无键连续执行。
- 时间处理：A/B 各自生成的 `created_at/updated_at` 在 HTTP body 比较时屏蔽；
  其余字段逐值比较，数据库与重启测试另验实际持久化。

## 5. 已确认的旧契约

- W02–W06 只要求请求 body 中 `keyword` 非空；路径 ID 存在时不校验显示文本是否匹配。
- W02–W06 路径 ID 不存在时，冻结旧 Flask 真实表现为未捕获异常后的 **500 HTML**；B 为迁移兼容保留这一行为。
- W08/W11 活动重复为 **400**，不是 409；409 只用于显式幂等键被不同输入复用。
- 已归档空组再次创建同名组是旧 API 可达路径：冻结 A 因
  `keyword_groups.label` 的 UNIQUE 约束抛出未捕获 `sqlite3.IntegrityError`，
  返回 Flask 默认 **500 HTML**，数据库仍只有原归档行；B 已精确保持相同
  status/body 和归档状态。
- W10 非空组为 **400**：`请先清空分组内关键词，再删除分组`；missing group 为 **404**。
- W13 missing keyword 为 **404**；W12 空 `keyword_text` 为 **400**：`关键词不能为空`。
- W14 manual 缺 days 为 **400**：`refresh_frequency_days is required for manual policy`。
- W09/W14/W15 的非法整数保留 Python `int()` 的旧错误文本。
- A/B 正常写响应除各自生成的 `created_at/updated_at` 外逐字段比较；错误响应不屏蔽字段。
