- **已通过**：远端调用、异步轮询、搜索结果计数、刷新任务成功、刷新历史可读取、远端 Markdown 未写入 Hub（Hub Markdown=0）、失败语义/隔离 DB 契约测试。

## 5. 风险与后续阻塞

远端服务需要在完成态增加每篇文章的 canonical URL（或提供可按 `saved_file`/结果项读取 URL 的正式接口）；仅凭标题不能推断 URL，不能为了通过门禁生成假 URL 或假 `content_id`。修复远端契约后，应先重跑两词对账，再执行 10 轮小规模门禁。

## 6. 前端状态契约修复

Hub `/api/refresh-all`、`/api/refresh-all/status`、`/api/refresh-all/history` 现在对外只输出 `completed`、`completed_with_failures`、`cancelled`、`failed`、`running`；内部枚举在 `hub_status` 保留。失败项为对象数组并带 `reason`，同时有 `failure_reasons`、`cancel_reason`、`snapshot_count`。`completed_with_failures` 的刷新 POST 返回 409，避免前端误判为全成功；没有 resume 路由/UI，本次未改旧只读前端。
