# 微信关键词原子切换与回退 v1

## 接口

`POST /api/v1/governance/switches/wechat-search/cutover`

```json
{
  "data_mode": "compare",
  "expected_mode": "legacy",
  "actor": "operator-name",
  "reason": "进入全量契约对账",
  "dry_run": false
}
```

- `data_mode`：仅允许 `legacy`、`compare`、`hub`。
- `expected_mode`：可选的乐观并发前置条件；任一读契约不匹配即返回 409，整批不变。
- `hub`：必须同时提供 `confirm: true` 和非空 `reason`。
- `dry_run: true` 或 `preflight: true`：执行完整清单、启用状态、模式一致性和 Hub-only 校验，但不更新、不写成功审计。

## 原子边界

- 一个 `BEGIN IMMEDIATE` SQLite 事务只更新原微信页面登记的 **22 个 GET contract**，成功响应固定 `changed_count=22`。
- 22 个读契约必须全部存在、`enabled=1`，且当前模式完全一致；提供 `expected_mode` 时还必须全部匹配。
- **21 个写入/外部 contract 固定为 `hub/hub`**。总开关不会更新它们，单契约治理接口也禁止将它们切回 legacy/compare。
- 缺失、禁用、状态漂移、expected_mode 不匹配、进程写锁或 SQLite 锁冲突均返回 409；更新和审计任一失败都会回滚整批。

## 审计与回退

成功切换写入 `audit_log`，action 为 `migration_switch.wechat_cutover`，包含 22 个 contract 的前后模式、actor、reason、expected_mode 和 changed_count。回退使用同一接口把 `data_mode` 设置为 `legacy`，建议显式填写当前 `expected_mode`，例如：

```json
{
  "data_mode": "legacy",
  "expected_mode": "hub",
  "actor": "operator-name",
  "reason": "异常回退"
}
```
