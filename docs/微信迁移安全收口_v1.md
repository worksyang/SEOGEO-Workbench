# 微信迁移安全收口

- `HUB_WECHAT_SOURCE_URL` 只允许本机隔离参考 `http://127.0.0.1:8774`；`localhost`/`::1` 会归一化，8765、外网、凭据、异常 scheme/port/path/query/fragment 均在启动配置阶段拒绝。
- `HUB_WECHAT_FREEZE_ROOT`（兼容 `HUB_WECHAT_SOURCE_ROOT`）必须位于项目 `data/migration/wechat/` 下，且不得指向 `/Users/works14/.claude/监控/wechat-ybxhyyh-top3` 真实旧源或其子目录。
- compare 会在 `contract_comparisons` 写入同事务 `audit_log` 证据：module、contract、请求指纹摘要、状态和 comparison_id；不写请求体或异常原文。
- `/api/v1/wechat/*` 已纳入 HTTP operation 登记：关键词详情复用 `keyword`，文章详情复用 `article-hit-detail`，单词刷新登记为 write `keywords-refresh`；旧 `/api/...` 43 个操作的登记与开关保持不变。
- v1 单词刷新由 middleware 强制 `keywords-refresh=hub`，路由直接调用 `WechatRefreshService`。必须显式 `confirm=true` 与 Idempotency-Key；默认 provider disabled 时持久化 command/job/item/event/audit/receipt 后返回真实 `409 blocked`，不调用 legacy adapter。
- v1 刷新状态在 `hub` 模式只读 SQLite 持久任务，未知任务返回 404，重启后仍可恢复；只有 `refresh-status` 开关明确启用为 `legacy/compare` 时，才允许读取配置硬限制的隔离参考 `127.0.0.1:8774`。
- v1 本地历史导入明确标记为 `hub_only=true, resolver=false`，不使用不存在的 migration switch，也不会因缺行静默落到 legacy HTTP。
