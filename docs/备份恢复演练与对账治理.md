# 备份 / 恢复演练与对账治理

## 范围

该闭环只操作工作台自己的 SQLite 运行库，默认目录为 `data/hub/backups/`；不会启动、修改或覆盖七套原始系统。备份创建使用 SQLite 在线备份 API，完成后执行 `integrity_check`，列表只标记真实通过校验的普通文件。

## API

- `GET /api/v1/governance/backups`：列出备份、大小、修改时间、SHA-256、完整性和可验证状态。
- `POST /api/v1/governance/backups`：创建在线备份。默认复用已有的同标签可验证备份；传 `{ "reuse": false }` 才会创建新文件。
- `POST /api/v1/governance/backups/{name}/restore-drill`：将备份恢复到 `data/hub/backups/restore-drills/` 隔离目录，仅校验恢复结果和表数量，绝不写入运行库。
- `GET /api/v1/governance/reconcile`：执行只读对账检查并把安全 JSON 报告写入 `data/hub/reports/reconcile/`；异常继续进入 `correction_jobs`，不直接修正事实。

所有创建和恢复演练会写入 `audit_log`。审计和 API 响应只输出备份文件名、相对路径、摘要哈希和状态，不输出 Cookie、Token、API Key 或其他敏感配置。

## 安全边界

备份名称必须是单一文件名；路径穿越、运行库路径、软链接和备份目录外路径都会被拒绝。恢复演练失败会清理临时文件和目标文件。该能力没有“覆盖恢复”接口，真实恢复前应先人工核对报告、备份哈希和运行库状态。
