## 0. 项目重新定向

### 0.1 旧计划的错误

v1 把项目描述成“一个新的 React 工作台”，并安排“Demo 视觉转换为 React 页面”。这会自然导向：

- 重新概括业务能力；
- 用通用卡片、表格和空态重做页面；
- 先完成 Hub 和 API，再补简化版 UI；
- 以接口存在、数据量和测试数代替产品保真；
- 把 WritingMoney 合并为一个写作页面；
- 把数据治理和系统状态提升为主导航。

该方向已被用户否定。现有 `workbench` 前端只作为技术原型，不再作为产品事实源。

### 0.2 v2 的正确目标

```text
统一首页和左侧导航来自 Demo
        +
每个业务岛屿来自原系统真实前端
        +
兼容 API 保持原交互契约
        +
v3.3 Hub、模块运行表和 Markdown 替换旧数据底层
```

### 0.3 不再使用的完成判断

以下证据单独存在时，不能声明模块完成：

- API 已返回；
- Hub 已有数据；
- 测试函数通过；
- 页面有 loading/empty/error；
- 页面能展示少量摘要；
- 路由能打开；
- Toast 显示成功；
- 原服务不可用时能诚实报错。

### 0.4 新的完成判断

模块只有同时达到下列条件才能完成：

1. 原页面结构完整迁入；
2. 原主要操作全部可执行；
3. 正常真实数据态达到原信息密度；
4. 数据路径逐步切换到新底座；
5. 新旧结果对账；
6. 异常态不伪造；
7. 主代理完成浏览器逐项验收。

## 4. 原页面复制协议

每个模块开发前必须先完成 provenance：

1. 列出原页面入口；
2. 列出所有源码文件；
3. 计算 SHA-256；
4. 保存原运行截图；
5. 列出所有按钮、筛选、排序、弹窗、抽屉和快捷键；
6. 列出所有 API；
7. 列出页面依赖的旧数据库、JSON 和 Markdown；
8. 明确允许修改项。

### 4.1 允许修改

- API base；
- 静态资源 base；
- 宿主挂载；
- CSS 作用域；
- 跨模块跳转；
- 安全问题；
- 绝对路径；
- 已批准的 Mac 兼容调整。

### 4.2 不允许修改

- 业务页面分栏；
- 一级和二级业务页签；
- 主要字段和指标；
- 核心按钮；
- 列表行信息；
- 图表；
- 详情层级；
- 用户已经习惯的操作顺序。

### 4.3 禁止的“简化”

- 用一个概览卡代替完整详情；
- 用下拉框代替原筛选组；
- 用文本说明代替趋势图；
- 只显示前 30 条而没有原列表行为；
- 把可编辑功能改成只读；
- 把刷新改成“稍后接入”；
- 因 Hub 没有字段而删除 UI。

Hub 缺字段时应补契约、兼容计算或模块运行表。

## 5. 技术方案

### 5.1 宿主

- React + TypeScript + Vite 可保留；
- 只负责全局外壳、路由和模块装载；
- 不重新编写业务岛屿；
- 模块之间通过显式跳转和 artifact envelope 通信。

### 5.2 模块

- 微信/XHS/Wiki/GEO：优先迁入原 HTML/JS/CSS；
- MP：保留原 Vue 组件和内部样式；
- WritingMoney：保留现有 HTML/JS 交互稿并后端化；
- 发布：以 Demo 页面为产品稿，把旧 Python 能力包装成命令服务。

### 5.3 后端

- FastAPI 继续作为统一入口；
- 模块 BFF 保持原响应结构；
- Repository 隐藏 Hub/运行表/旧系统差异；
- 写操作通过持久命令；
- 所有真实操作保留原始错误。

### 5.4 数据库

- 默认一个 SQLite 文件、多个逻辑层；
- 14 张跨系统核心表不混入模块 UI 状态；
- 模块运行表使用清晰前缀；
- WAL、`BEGIN IMMEDIATE`、进程锁；
- 备份、恢复演练和对账。

## 15. 数据迁移计划

### 15.1 迁移单位

不再按“全库完成后再做页面”，而是按模块纵切：

```text
页面冻结
→ 页面迁入
→ 兼容 API
→ 历史导入
→ 单接口切换
→ 写操作双写
→ 对账
→ 接管
```

### 15.2 历史导入要求

- manifest；
- file hash；
- source_ref；
- accepted/rejected；
- checkpoint；
- 幂等重放；
- 记录数对账；
- 内容身份冲突；
- 原始文件不变。

### 15.3 写入切换

每个写操作分别切换：

- 关键词设置；
- 刷新任务；
- MP 账号/分类；
- Wiki 保存；
- WritingMoney 项目/批次；
- 发布队列。

禁止一次性替换全部写路径。

### 15.4 回退

每个接口拥有：

- `legacy`；
- `compare`；
- `hub`；

三种运行模式。发现差异时立即回到 `legacy/compare`，不修改原历史。

## 18. 返工和偏移门禁

出现以下任一情况立即暂停开发：

- 新页面主要结构与原系统不一致；
- 原系统存在的操作未进入功能清单；
- 为适配 Hub 删除字段；
- 正常态未完成就大量制作错误态；
- 用测试数量代替浏览器证据；
- 子代理擅自重构页面；
- Demo 主导航被改变；
- 母文章铸造和批量成稿重新合并；
- 真实发布在未确认时被触发。

暂停后必须：

1. 保存差异截图；
2. 标记偏移原因；
3. 回到原页面源码；
4. 修订基线；
5. 主代理批准后继续。
