# 全域内容工作台验收矩阵 v2（T001–T300）

> **状态**：已获批准，执行中（2026-07-15）；未通过项仍不得宣称完成。
> **架构依据**：`全域内容资产与观测架构方案_v3.3.md`。
> **开发依据**：`全域内容工作台开发总计划_v2.md`。
> **产品依据**：`unified-content-platform-demo.html` 与七套原系统真实页面、源码和运行状态。
> **验收目标**：证明新工作台既保留 Demo 外壳和原系统完整操作，又完成 v3.3 数据底座、迁移、审计、备份与恢复。
> **硬性结论**：**T001–T300 必须全部 PASS；BLOCKED、SKIP、只有自动化结果或只有 Toast 均不算通过。**

## 1. 验收口径

### 1.1 三类事实分别验收

| 层 | 验收事实源 | 主要判断 |
|---|---|---|
| 统一外壳 | Demo | 导航、配色、页面边界、首页和跨模块跳转一致 |
| 业务岛屿 | 原系统 | 页面结构、信息密度、字段、按钮和操作顺序完整保留 |
| 数据底座 | v3.3 | 身份、快照、指标、任务、资产、迁移、审计和恢复正确 |

### 1.2 正常态优先

每个模块必须按以下顺序验收：

1. 真实历史数据正常展示；
2. 原版读取、搜索、筛选、排序和详情操作；
3. 可逆写操作及真实落盘；
4. 持久任务、进度、取消和恢复；
5. 空态、错误态、离线态和权限阻断；
6. 新旧数据对账和回退。

空态、错误态或模拟数据不能代替正常业务态。Fake Provider 只允许用于 AI 生成、GEO 计费刷新和真实发布等外部或不可逆操作的自动化安全测试。

### 1.3 PASS 规则

每项用例只能记录：

- `PASS`：步骤完成、通过标准满足、证据齐全；
- `FAIL`：行为或结果不符合标准；
- `BLOCKED`：受登录、外部服务、计费授权或环境影响无法执行；
- `NOT RUN`：尚未执行。

只有 `PASS` 计入完成率。外部条件阻塞必须保留真实原因，不得用模拟成功替代首版完成。

### 1.4 优先级

- `P0`：方向、核心操作、数据安全或不可逆动作；任一失败即停止交付；
- `P1`：原系统主要功能和迁移正确性；首版必须通过；
- `P2`：兼容性、体验和可维护性；本矩阵中同样必须通过。

## 2. 证据规范

| 代码 | 证据 |
|---|---|
| `B` | 原系统或 Demo 基线截图、功能清单、源码位置与 hash |
| `N` | 新工作台同状态截图，包含 URL、时间、数据范围 |
| `C` | Codex 主代理使用内置浏览器完成的逐步点击记录 |
| `A` | 请求参数、API/Command 回执、状态码和错误体 |
| `D` | SQLite 查询、Markdown/文件 diff、manifest 或 hash |
| `L` | job event、audit log、错误日志和重试记录 |
| `R` | 新旧对账、备份恢复、性能或安全测试报告 |

所有涉及页面的用例至少需要 `N+C`；保真用例需要 `B+N+C`；写操作需要 `C+A+D+L`；迁移与恢复需要 `D+R`。截图不能只截局部成功提示，必须能定位页面上下文。

## 3. 执行环境与总门禁

- 统一入口：`http://127.0.0.1:8799/`；
- 默认只监听 `127.0.0.1`，不得暴露到局域网或公网；
- 主视觉基准：1440×1000；兼容尺寸：1280×720、1440×900、1920×1080；
- 页面最终验收只使用 Codex 内置浏览器；
- 自动化测试可辅助定位问题，但不能替代主代理点击验收；
- 原系统、`source/**`、`source/_local_full_backup/**` 与 Demo 永久只读；
- 公众号首版必须完成至少一次受控的真实账号抓取；
- GEO 计费刷新和真实群发默认只测 dry-run/草稿；只有用户显式确认后才执行真实动作；
- 每个模块先通过原 UI 正常态，再切换数据接口；不得用后端完成度抵扣前端缺失。

## 17. 最终完成定义

首版只有同时满足以下条件才算完成：

1. T001–T300 全部为 `PASS`；
2. 九个页面在统一外壳内可进入，八个业务入口拥有正常真实数据态；
3. 原系统主要页面结构和操作全部保留，没有“摘要版替代”；
4. 公众号完成至少一次受控真实抓取；
5. AI、GEO 和发布的不可逆动作遵守确认门，自动化不真实群发；
6. 14 张核心表、模块运行表、Markdown 和 raw 事实边界正确；
7. 历史导入可幂等、可对账、可回退；
8. 备份在隔离目录恢复成功；
9. 零已知 P0/P1；
10. Codex 主代理亲自完成最终内置浏览器逐项验收。

> **验收的核心不是“新系统有多少功能”，而是“原来能做的事在统一工作台里仍然能做，并且每一次读取、修改、任务和产物都获得更可靠的数据底座”。**
